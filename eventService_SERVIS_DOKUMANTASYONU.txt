================================================================================
EVENT SERVICE SERVİSİ - CLASS VE METOD DOKÜMANTASYONU
================================================================================

Bu servis, etkinlik (event) yönetimi ile ilgili tüm işlemleri yapar.

--------------------------------------------------------------------------------
1. EventServiceApplication
--------------------------------------------------------------------------------
AÇIKLAMA: Spring Boot uygulamasının ana başlangıç sınıfıdır.
İŞLEVİ: 
  - Uygulamayı başlatır
  - Feign Client desteğini aktif eder (diğer servislere HTTP çağrıları için)

METODLAR:
  - main(String[] args): Uygulamayı başlatır

NOTLAR:
  - Port: 8085
  - @EnableFeignClients: Feign Client kullanımını aktif eder

--------------------------------------------------------------------------------
2. EventController
--------------------------------------------------------------------------------
AÇIKLAMA: Event ile ilgili HTTP endpoint'lerini sağlar.
İŞLEVİ: REST API endpoint'leri sunar

METODLAR:
  - create(EventReq request): Yeni bir etkinlik oluşturur (POST /api/events)
  - getById(String id): ID'ye göre etkinlik getirir (GET /api/events/{id})
  - getAll(): Tüm etkinlikleri getirir (GET /api/events)
  - delete(String id): Etkinliği siler (DELETE /api/events/{id})
  - publish(String id): Etkinliği yayınlar ve Kafka'ya EVENT_CREATED event'i gönderir (POST /api/events/{id}/publish)
  - openSales(String id): Etkinlik satışlarını açar (POST /api/events/{id}/open-sales)

--------------------------------------------------------------------------------
3. EventService (Interface)
--------------------------------------------------------------------------------
AÇIKLAMA: Event işlemleri için servis arayüzü
İŞLEVİ: Event işlemlerinin metod imzalarını tanımlar

METODLAR:
  - create(EventReq request): Etkinlik oluşturma
  - getById(String id): ID'ye göre etkinlik getirme
  - getEntityById(String id): Entity olarak etkinlik getirme (iç kullanım için)
  - getAll(): Tüm etkinlikleri getirme
  - delete(String id): Etkinlik silme
  - publish(String id): Etkinlik yayınlama
  - openSales(String id): Satışları açma

--------------------------------------------------------------------------------
4. EventServiceImpl
--------------------------------------------------------------------------------
AÇIKLAMA: EventService interface'inin implementasyonu
İŞLEVİ: Event işlemlerinin gerçek implementasyonunu yapar

METODLAR:
  - create(EventReq request): 
      * EventReq'den Event entity'sine dönüşüm yapar
      * Venue bilgisini yükler
      * Status'ü DRAFT olarak ayarlar
      * Veritabanına kaydeder
  
  - getById(String id): 
      * ID'ye göre etkinlik bulur
      * EventRes DTO'suna dönüştürür
  
  - getEntityById(String id): 
      * Entity olarak döner (diğer servisler için)
  
  - getAll(): 
      * Tüm etkinlikleri getirir ve DTO'ya dönüştürür
  
  - delete(String id): 
      * Etkinliği siler
  
  - publish(String id): 
      * Etkinliğin status'ünü PUBLISHED yapar
      * Kafka'ya EVENT_CREATED event'i gönderir
      * Bu event Ticket Service tarafından dinlenir ve bilet stoku oluşturulur
  
  - openSales(String id): 
      * Satış başlangıç tarihini kontrol eder
      * Status'ü SALES_OPEN yapar
  
  - mapToResponse(Event event): 
      * Event entity'sini EventRes DTO'suna dönüştürür
  
  - mapToEvent(EventReq request): 
      * EventReq DTO'sunu Event entity'sine dönüştürür
  
  - publishEventCreated(Event event): 
      * Kafka'ya EVENT_CREATED event'i gönderir
      * Event ID, Venue ID ve Price Category detaylarını içerir

17. Event (Entity)
--------------------------------------------------------------------------------
AÇIKLAMA: Etkinlik veritabanı entity'si
İŞLEVİ: Etkinlik bilgilerini veritabanında saklar

ALANLAR:
  - id: Etkinlik ID'si (UUID)
  - name: Etkinlik adı
  - description: Açıklama
  - imageUrl: Görsel URL'i
  - eventDate: Etkinlik tarihi
  - doorsOpen: Kapı açılış saati
  - salesStartDate: Satış başlangıç tarihi
  - salesEndDate: Satış bitiş tarihi
  - status: Etkinlik durumu (DRAFT, PUBLISHED, SALES_OPEN, vb.)
  - attributes: JSON formatında ek özellikler
  - venue: Mekan bilgisi (ManyToOne ilişki)
  - priceCategories: Fiyat kategorileri (OneToMany ilişki)
